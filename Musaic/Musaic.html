Musaic

Musaic allows users to create a meaningful art piece displaying their musical tastes through a personal mosaic.

For Musaic, I worked with my friend Ram as the UI Designer, UX Designer, and Front End Developer to create the website

Duration
Sept 2019 - Jan 2020
Role
UX Design, UI Design, Front End Development
Tools
Sketch, Invision, HTML, CSS, Javascript, Pug/Jade
Platform
Website
Team
2

Inspiration

I was taking an information studies course over the 2019 summer break and we had to make hypertext link portrait, which was something that was created through HTML and CSS uniquely representing us. As someone who loves music, I decided to create a face composed of my favorite albums of all time (starting from the top). You can check mine out here.

I enjoyed making this "portrait," but it was incredibly tedious to do, album by album. I knew there must be a better way to directly extract album covers from popular streaming services and make differnt mosaics out of them.

The Pitch

Right before the start of the 2019 Fall Quarter at UCLA, I contacted my good friend Ram with a more fleshed out version of my intial idea. I Ram is a computer science major at UCLA who I knew would be able to help me bring my idea to life. As seen above, I sent him a custom "pitch" of my idea to create a website/application (at the time) that would connect the user's streaming service (Ram uses Spotify, I use Apple Music), ask the user for a photo (that's Ram above), select the tiles of the mosaic from some selection menu, and return the resulting mosaic picture.

Ram resoponded "Dang this is sick haha...Love this."

I sucessfully recruited my backend developer for this project.

Intial User Flow

The initial user flow included the user going to the homepage and having two options:
1. Creating their own Musaic
2. Exploring the gallery of Musaic's on the homepage

For creating the Musaic, the user would go through three steps:
1. Signing into their Spotify account
2. Uploading a personal photo as the backdrop
3. Selecting a playlist from your Spotify account

The reasoning behind having the order of these three steps was to have the users realize right away that they needed a Spotify account instead of learning they needed a Spotify account after uploading a personal photo. The upload of the personal photo was chosen to be before selecting a playlist because visually, we want users to create a visual hierarchy of laying the "foundation" of their Musaic (the personal photo) first and the adding the tiles (the album covers) afterwards.

Grayscale Wireframe (Lo-Fi)

After creating the intial user flow, I had to visualize what this user flow would look like in webpages. Below is what the inital HTML pages looked like.

There are a few changs from the grayscale wireframe to the HTML pages:

The vertical positioning of the visualizations in the wireframe to the horizontal positioning in the HTML pages. This change was made to create more space below for other items more visible such as provided photos (more about that below) and playlists.

The playlist selection page in the HTML pages. Ram and I agreed that not only is it easier to program the other confirmation page, but having one page dedicated to confirming the playlist would require the focus of the user more than an overlay on the same page, therefore reducing mistakes in generating Musaics.
